# 什么是 Markdown

Markdown 是一种轻量级标记语言，它允许使用者使用纯文本格式编写文档，通过简单的特殊符号来替代排版样式。文档可被转换为多种格式，如 HTML、PDF 和 Word 等。

## 为什么要使用 Markdown？ 

有观点认为“使用特殊符号表示格式”过于奇怪，或者它只是程序员的工具。但在现代数字写作中，它是公认最高效、最通用的文本方案：

- **专注内容创作**：无需在编写文档时频繁将手移开键盘去点击鼠标调整字号、粗细和颜色。只需专注于文字内容，排版即可随手完成。 
- **极简的学习曲线**：只需掌握几个核心符号（如 `#`、`-`、`**`）即可无门槛上手。 
- **轻量与跨平台**：文件实质为纯文本格式，体积极小。任何文本编辑器（如系统自带的记事本）都能直接打开、阅读并编辑内容。 
- **兼容性强与格式转换**：可无缝渲染成网页（HTML），或利用导出工具生成排版完美的 PDF、Word 等格式。
- **节约大模型成本**：使用 Markdown 编写的内容不含多余的样式代码，可在消耗最少 Token 的情况下被 AI 准确理解与处理。
- **与可视化不冲突**：在很多先进的编辑器里，Markdown 也可以使用“所见即所得”的实时预览方式进行可视化编辑，兼顾效率与直观。

## 为什么不直接使用 Microsoft Word 或 HTML？

### ❌ Microsoft Word 的局限性

- **结构不透明**：Word 文档（`.docx`）本质上是一个样式、文字、图片混合在一起的 XML 压缩包。想要直接通过脚本或外部程序快速提取出纯净的内容非常复杂。
- **兼容性较弱**：同一份 Word 文档，在 Microsoft Office、WPS、LibreOffice 以及手机端打开时，经常会出现表格变形等样式不完全相同的问题。
- **存储开销大**：因为其中存在大量的元数据 XML 代码，导致即使只写了 1 个字符的 Word 文档，其底层体积也需要 13KB 左右，不便于海量存储和轻量分发。

### ❌ HTML 的局限性

- **存在隐私与安全风险**：HTML 文档通常允许直接夹带 CSS 样式、甚至 JavaScript 脚本。在本地打开未知的 HTML 时，可能会在后台触发未授权的远程网络请求（例如加载远程图片），从而泄露用户的真实 IP 或其他隐私信息。
- **内容过于臃肿**：HTML 包含大量重复闭合的标签（如 `<div>`、`<p>`），语法结构冗长，在没有可视化辅助工具的情况下，极难进行纯手写输入和人类直观阅读。
